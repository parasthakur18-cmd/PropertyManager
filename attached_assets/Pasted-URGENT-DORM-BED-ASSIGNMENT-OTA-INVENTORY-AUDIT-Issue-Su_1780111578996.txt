URGENT DORM BED ASSIGNMENT & OTA INVENTORY AUDIT

Issue Summary:

Dorm bookings with multiple guests are not remaining allocated correctly after confirmation.

Example:

Booking Name: Test
Property: Yolo Hostel
Dates: 1 Jun – 2 Jun
Guest Count: 4 Guests
Room Type: 4-Bed Dorm

Current Behavior:

1. When the booking is first created, the calendar temporarily shows all 4 beds occupied.
2. The booking details correctly show 4 guests.
3. After changing status to Confirmed, only 1 bed remains occupied in the calendar.
4. Availability becomes incorrect.
5. Dorm inventory becomes incorrect because only 1 bed is counted as occupied.
6. The booking still shows 4 guests in reservation details.

Expected Behavior:

For a booking with 4 guests in a 4-bed dorm:

Bed 1 → Test
Bed 2 → Test
Bed 3 → Test
Bed 4 → Test

OR

4 separate bed allocations linked to the same booking.

Calendar should show:
- 0 beds available
- 4 beds occupied

Inventory sync should push:
- Remaining beds = Total beds - 4

ADDITIONAL OTA ISSUE
--------------------------------------------------

When a booking comes from an OTA (Booking.com, Hostelworld, Goibibo, MakeMyTrip, Airbnb, etc.):

Example:

Dorm Capacity = 4 Beds

OTA Booking:
- Guests = 3
- Room Type = Dorm Bed
- Dates = 1 Jun – 2 Jun

Current Behavior:

- Booking is created for 3 guests.
- Calendar blocks only 1 bed.
- Inventory counts only 1 occupied bed.
- Remaining availability becomes incorrect.
- Overbooking risk is created.

Expected Behavior:

OTA booking with 3 guests must:

- Allocate 3 separate beds.
- Block 3 beds in calendar.
- Reduce inventory by 3 beds.
- Sync correct remaining availability to all channels.

Example:

Dorm Capacity = 4 Beds

OTA Booking = 3 Guests

Calendar:

Bed 1 → Guest
Bed 2 → Guest
Bed 3 → Guest
Bed 4 → Free

Availability:

1 Free
3 Occupied

Inventory Push:

Remaining Beds = 1

This rule must work for:
- Direct bookings
- Walk-ins
- OTA bookings
- Imported reservations
- API reservations
- Group bookings
